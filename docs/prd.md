# 부산 AI·DX 지원사업 문의 포털 — PRD

**버전:** v1.0
**작성일:** 2026-05-26
**대상 산출물:** `busan-dx-portal.html` (단일 HTML, Tailwind CDN)

---

## ① 목적

부산정보산업진흥원(BIPA) 지원사업 문의의 **접수→답변 평균 응답 시간**을 단축한다.

- 측정 지표: 평균 응답 시간 **영업일 2일 이내**
- 현재 분산된 전화·이메일 문의 채널을 단일 웹 창구로 통합하지 않고, 응답 사이클 자체를 줄이는 데 집중

---

## ② 사용자

| 그룹 | 설명 | 사용 화면 |
|---|---|---|
| **일반 문의자** | 지역기업 담당자 / 시민 / 예비창업자 | 화면 1 |
| **BIPA 담당자(관리자)** | 문의 접수 검토·답변 작성·상태 관리 | 화면 2 |

비고: BIPA 매니저(통계 감독자) 등 추가 역할은 v1 범위 밖.

---

## ③ 핵심 기능 (5)

1. **문의 등록 + 빈 값 검증**
   - 7카테고리 드롭다운 + 제목·이름·내용 입력
   - JS 유효성 검사("모든 항목을 입력해주세요"), 브라우저 기본 검증(`required`)은 비활성화
   - 접수 시 자동으로 상태 = `접수`

2. **카테고리(7) + 상태(4) 필터 칩**
   - 한 줄 가로 스크롤 (`flex-nowrap` + `overflow-x: auto`)
   - 선택된 칩만 짙은 파랑 배경 + 흰 글자, 나머지는 회색 보더만
   - 카테고리·상태 동시 필터링 가능

3. **상태 배지 + 상대시간 자동 갱신**
   - 4상태 배지 색상 구분(파랑 계열 단조, ⑤ 참고)
   - "방금 / 3분 전 / 1시간 전 / 2일 전" 형식
   - `setInterval` 30초 간격 자동 갱신

4. **localStorage 영속화**
   - 키: `bipa-portal:inquiries`
   - 등록·삭제·상태변경마다 저장 → 새로고침 후에도 유지
   - 최초 진입 시 샘플 5건 시드 후 저장, 모두 삭제한 상태도 보존(재시드 X)

5. **관리자 모드: 답변·상태 변경** (화면 2, 다음 Plan Mode 실습 구현)
   - 4상태 전환 (접수 ↔ 검토중 ↔ 답변완료 ↔ 보류)
   - 답변 텍스트 작성 (문의별 1개)
   - 답변 작성 시 자동으로 상태 = `답변완료`로 전환 옵션

## ④ 화면 (2)

### 화면 1: 일반 문의자 페이지
- 헤더 (제목 + 부제)
- 문의 작성 폼 카드 (상단)
- 필터 칩 행 (카테고리 7 + 상태 4)
- 문의 목록 (최근순): 카테고리 배지 + 상태 배지 + 제목 + 작성자 + 상대시간 + 본문 + 삭제 버튼

### 화면 2: 관리자 모드 (다음 실습 구현 예정)
- 화면 1과 동일한 목록 구조
- 각 항목 우측에 **[상태 변경 ▾] [답변 작성]** 액션 추가
- 답변 작성 시 인라인 textarea + "저장" 버튼
- 화면 1↔2 전환: 우상단 토글 ("관리자 모드" 버튼)
- 측정: 화면 전환 클릭 수 주당 1회 미만 (관리자는 화면 2 기본 진입)
